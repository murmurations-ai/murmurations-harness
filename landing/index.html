01PhilosophySource as a human role

The human holds the vision. The harness holds the flock.

Most agent frameworks treat the human as a configuration step. The Murmuration Harness treats the human as Source — the person who initiated the creative impulse, holds the vision, and bears accountability for the work.

Source is not a configuration. It is a role. It cannot be delegated, optimized away, or rotated. The concept comes from Peter Koenig and Tom Nixon's work on creative authorship in collaborative organizations.

Think of it as a suit of power armor. The human provides direction, judgment, and values. The harness provides strength, reach, and tirelessness. Neither is useful without the other.

It is not an autonomous agent framework.
It is a tool that amplifies human agency.

02How it worksFour moving parts
/01

Agents wake, then decide.

Each agent has a role.md declaring its identity, schedule, signal scopes, and write surface. On every wake the daemon assembles a signal bundle — GitHub issues you've labeled for them, private notes, governance events — and the agent reads what changed, decides what to do, and produces structured actions. Idle wakes are skipped when nothing has changed; bundle load is surfaced as a [b:N] badge in the dashboard.

/02

Governance is pluggable.

Five governance models ship bundled — S3 sociocracy, Chain of Command, Meritocratic, Consensus, Parliamentary. Or write your own plugin and load it by path. The core harness contains no S3-specific terms; consent rounds, objections, and ratification live in the plugin. Switch models by changing one line in harness.yaml.

/03

GitHub is the system of record.

Every action item, every meeting outcome, every directive — written to GitHub Issues. Operators reading gh issue list see the same truth the agents do. Local files capture only ephemeral runtime state (.murmuration/) and durable identity (agents/, governance/). Nothing important hides on one machine.

/04

The dashboard shows what's true now.

A read-only TUI panel surfaces every agent's last wake, productivity status, cost burn, signal-bundle load, and obligation results. Badges like [obl-unmet:2] and [b:23] flag drift the second it happens. murmuration doctor diagnoses setup; list-stale-issues inventories the issues bloating per-wake context.

03QuickstartFive minutes

From install
to first meeting.

The harness ships an interactive scaffolder. murmuration init writes a working configuration, a sample agent, and a starter .env.example. doctor validates the setup before the daemon starts.

If you already have a CLI subscription (Claude Code Pro/Max, ChatGPT, or Google), the scaffolder auto-detects it. Pick "subscription-cli" at the prompt and skip the API key. $0 marginal cost.

Otherwise paste a free-tier Gemini key. Either path lands at the same first meeting.

Security Threat Taxonomy → Harness Mitigations
Environment Injection
GitHub issue bodies are adversarial by default. Mitigation: untrusted signal classification · sanitizer/renderer · no tool grants from untrusted text.
Memory Attacks · Skill Poisoning
Single-turn injection → persistent malicious skill (Hermes gap). Mitigation: semi-trusted memory/skills segments · RunLedger signed provenance · Phase 6 security scan on write.
Toolchain Abuse · Dangerous Compositions
Innocent tools chained into harm (read + send = exfiltrate). Mitigation: deny-by-default · ToolGrant allowlists · ToolInvocationRecorder tracks call sequence · composite allowedSideEffects check.
Agent Network Attacks
Cross-agent directive injection (harness#353/354). Mitigation: per-agent routing filter in DefaultSignalAggregator · agentId-keyed isolation · GovernancePlugin tension protocol.
Prompt Abuse
Directive overrides · jailbreak · instruction hierarchy bypass. Mitigation: trust propagation rules — untrusted/semi-trusted text cannot alter policy or grant permissions. ADR-003X.
Multi-Agent Memory Poisoning
Shared GitHub repo as signal channel — one poisoned directive affects all agents. Mitigation: routing isolation (security primitive, not just correctness) · RunLedger forensic trace.
Research Convergence — Three Independent Derivations of AgentRuntime
Barry Zhang / Anthropic
Practitioner derivation from production patterns at scale. Atomic wake unit = model + context + tools + environment + contracts + ledger.
Tsinghua NLAH
Academic derivation from formal agent evaluation. Five contract elements (inputs, budgets, permissions, completion conditions, outputs) map directly onto the same decomposition.
arXiv 2512.12791
"Beyond Task Completion" evaluation framework. 4-pillar (LLM · Memory · Tools · Environment) independently maps to the same four-layer structure. Empirical finding: 100% completion / 33% policy adherence.

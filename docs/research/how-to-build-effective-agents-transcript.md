**Context:** Keynote on agent design, simplicity, context management, and the future of multi-agent systems from the AI Engineer Summit.

## Executive Summary

In this keynote, Barry Zhang from Anthropic shares three core principles for building effective AI agents:

1. **Don't build agents for everything.** Agents thrive in ambiguous problem spaces. If a task has a strict decision tree, use a predictable, cheap workflow. Agents are expensive (in tokens/latency) and should be reserved for high-value tasks where you can tolerate errors or easily verify outputs (like coding).
2. **Keep it simple.** An agent is just a Model + Tools + Environment + System Prompt in a loop. Do not add complex orchestration until you have perfected this basic loop.
3. **Think like your agent.** Developers must put themselves in the agent's context window. An agent operates "in the dark," only seeing what is explicitly passed in its limited 10k-20k token window. If a human couldn't complete the task with _only_ the text in that window, neither can the agent.

**Future Horizons:** Zhang predicts the rise of budget-aware agents, self-evolving tools, and asynchronous multi-agent collaboration.
